Section 01

What Is StatementScrub?

StatementScrub is a SaaS underwriting CRM that combines AI-powered bank statement analysis with a smart lender matching engine. It replaces 45 minutes of manual review with a 30-second AI report — then instantly shows which of 435 lenders will fund the merchant.

🏦 The Problem It Solves

MCA brokers, mortgage underwriters, and business loan consultants waste hours manually reading bank statements — counting NSFs, totaling deposits, checking balances, and then calling each lender to see if they'll fund. StatementScrub automates all of it.

⚡ The Solution

Upload a bank statement PDF → Claude AI reads every transaction → generates a structured risk report in ~30 seconds → then matches the merchant against 435 lenders filtered by industry, state, deposits, NSFs, credit score, and positions.

👥 Who Uses It

MCA brokers submitting deals to funders, mortgage brokers verifying self-employed income, business loan consultants, SBA lenders, and alternative finance ISO reps who process 10–500 statements per month.

💰 How It Makes Money

Credit-based pricing ($1.49–$0.10 per analysis depending on pack size) plus monthly subscriptions ($49/mo for 50 analyses, $149/mo for 250). Stripe checkout is live. Free tier: 3 analyses.

Two Ways to Match

👥 Merchant Mode

Select any previously analyzed merchant. Deposits, balance, and NSF count are auto-filled from bank statement averages. User only needs to add industry, state, time in business. One click → results.

✏️ Manual Mode

Enter all financial criteria manually. Useful for pre-screening a merchant before uploading statements, or matching based on client-provided data without an uploaded PDF.

Results Display

  • Lenders grouped by grade: A (best) → B → C → D (last resort)
  • Each card shows: name, code, email (clickable), phone, ISO rep, notes, portal link
  • Max advance amount displayed per lender
  • Consolidation OK / Default History OK tags per lender
  • When no lenders match: shows WHY each lender declined (which criteria failed)
  • "Find Lenders" button on every merchant page — pre-fills from statement data

Section 08

Growth Gaps — What's Missing

These are the gaps between the current state and what's needed to acquire and retain paying users at scale.

🔴 Critical (Blocking Users)

GapImpactFix
No OCR for scanned PDFs ~40% of bank statements are scanned images — these return blank/gibberish text and fail silently Add Google Vision API or Tesseract OCR as fallback
No onboarding flow New users see an empty dashboard with no guidance — high immediate churn Add welcome wizard: upload first statement → see sample result → buy credits
SQLite in production SQLite doesn't scale beyond 1 server, no concurrent writes, no backups Migrate to PostgreSQL (AWS RDS or Supabase)
No error messaging on PDF fail When analysis fails (bad PDF, OCR fail), user sees generic 500 error Return specific user-friendly error: "Could not read this PDF — try another format"
App not persisted on reboot If server reboots, uvicorn dies and site goes down Add systemd service or PM2 process manager

🟡 High Priority (Retention & Revenue)

GapImpactFix
No team/multi-user accounts MCA shops have 3–15 reps — no way to share an account or have sub-users Add organization model with seats/roles (owner, analyst, viewer)
No saved lender lists / favorites Users find good lenders but can't save them — have to re-run match every time Add "Save Lender" to user account with notes per lender
No deal/application tracker Brokers need to track which merchants were submitted to which lenders and the outcome Add "Create Deal" from merchant page → track lender, amount, status
No re-analysis / update Users upload the same merchant monthly — no way to compare old vs. new Add "View Trend" comparing 3-month and 6-month deposit/balance history
No white-label / PDF branded report Brokers want to send reports to clients with their own branding Add logo upload + custom color → branded PDF export

🔵 Growth (Acquisition)

GapImpact
No affiliate/referral programMCA community is relationship-driven — brokers refer other brokers. No referral system = missing viral loop.
No Google/LinkedIn loginSignup friction increases drop-off. OAuth reduces it significantly.
No public API docsAPI key is generated but there's no documentation page — tech-savvy users can't self-serve.
No email nurture sequenceFree users who don't convert get no follow-up emails. Automated drip would recover churned trials.
